# 论坛体小说生成器 - Web应用使用说明

## 🚀 快速开始

### 1. 启动服务

```bash
cd /path/to/your/project
python src/main.py -m http -p 5000
```

### 2. 访问Web界面

在浏览器中打开：

```
http://localhost:5000
```

或者直接访问：

```
http://localhost:5000/assets/index.html
```

## 📱 功能说明

### 界面功能

1. **主题输入框**
   - 输入你想要的小说主题
   - 可以点击下方的示例标签快速填入

2. **示例主题**
   - 不体面的事
   - 图书馆暗恋
   - 复仇恋爱
   - 带血斧头
   - （点击标签自动填入）

3. **生成按钮**
   - 点击"✨ 生成小说"按钮开始生成
   - 生成过程中显示加载动画

4. **结果操作**
   - 📋 **复制** - 一键复制小说内容到剪贴板
   - 💾 **下载** - 将小说下载为txt文件

### 推荐主题格式

```
[事件描述]，[人物关系/故事类型]
```

**示例**：
- 我好像做了一件很不体面的事
- 我在图书馆遇到了暗恋的学长
- 室友昨天晚上带回一个快递，拆开里面是一把带血的斧头
- 我是为了报复才接近他的，结果自己栽进去了
- 我昨天晚上做了一件坏事

## 💡 使用技巧

### 1. 主题建议

- **要有冲突**：让故事有矛盾点（如：做坏事、暗恋、复仇）
- **明确关系**：说明人物关系（如：朋友、暗恋对象、死对头）
- **类型可选**：可以指定小说类型（如：悬疑、复仇、甜宠）

### 2. 结局类型

系统会自动选择合适的结局类型：
- 甜宠HE（大团圆）
- 悲剧BE（遗憾、失败）
- 开放式（留白）
- 悬疑反转
- 现实向
- 成长向
- 遗憾美
- 暧昧向
- 惊悚向
- 黑暗向
- 温情向
- ... 等等

### 3. 生成的小说特点

- ✅ 吸睛标题（5-20字，有悬念）
- ✅ 完整楼层（2-41楼连续）
- ✅ 楼主回复分散（11楼、21楼、31楼、41楼）
- ✅ 结局内容分散（不挤在一楼）
- ✅ 网友类型丰富（吃瓜群众、嗑学家、课代表等）
- ✅ 内容质量高（每条回复都有价值）

## 📝 注意事项

1. **服务端口**
   - 默认端口：5000
   - 修改端口：`python src/main.py -m http -p 8080`

2. **访问方式**
   - 本地访问：`http://localhost:5000`
   - 局域网访问：`http://你的IP:5000`（如：`http://192.168.1.100:5000`）

3. **生成时间**
   - 每次生成大约需要10-30秒
   - 请耐心等待加载完成

4. **浏览器兼容**
   - Chrome/Edge（推荐）
   - Firefox
   - Safari
   - 移动端浏览器

## 🌐 部署到外网

如果你想让朋友也能使用，可以：

### 方式1：使用ngrok（推荐）
```bash
# 安装ngrok
brew install ngrok  # Mac
# 或从 https://ngrok.com 下载

# 启动服务
python src/main.py -m http -p 5000

# 新开一个终端，映射端口
ngrok http 5000

# ngrok会给你一个外网访问地址，如：
# https://abc123.ngrok.io
```

### 方式2：云服务器部署
```bash
# 在云服务器上启动服务
nohup python src/main.py -m http -p 5000 > /dev/null 2>&1 &

# 访问地址
http://你的云服务器IP:5000
```

### 方式3：Docker部署（需要自行编写Dockerfile）
```dockerfile
FROM python:3.9-slim
WORKDIR /app
COPY . .
RUN pip install -r requirements.txt
EXPOSE 5000
CMD ["python", "src/main.py", "-m", "http", "-p", "5000"]
```

## 🆘 常见问题

**Q: 生成失败怎么办？**
A: 检查网络连接，确保API密钥配置正确，查看日志：`tail -f /app/work/logs/bypass/app.log`

**Q: 如何修改生成的小说长度？**
A: 修改 `config/` 目录下的JSON配置文件中的 `max_completion_tokens` 参数

**Q: 可以自定义结局类型吗？**
A: 可以！在主题中直接说明想要的结局类型，如："HE结局"、"BE结局"、"开放式结局"

**Q: 生成的小说可以直接发布吗？**
A: 建议先人工审阅和修改，确保内容符合发布平台的要求
